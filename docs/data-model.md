# 数据模型草案

本 demo 先使用 `services/mock-db.js` 作为本地数据源。正式上线时建议迁移为微信云开发数据库集合，或独立后端数据库表。

## 账号与绑定

| 集合/表 | 用途 |
|---|---|
| `users` | 微信用户、手机号、角色、绑定状态 |
| `invite_codes` | 教务生成的邀请码、角色、有效期、绑定目标 |
| `students` | 学生基础信息、所属班级 |
| `guardians` | 家长信息、手机号、绑定学生 |
| `teachers` | 老师信息、学科、授课班级 |

## 教务与课表

| 集合/表 | 用途 |
|---|---|
| `campuses` | 校区信息 |
| `classrooms` | 教室、摄像头状态、推流标识 |
| `classes` | 班级、老师、学生数量 |
| `courses` | 课程定义 |
| `schedules` | 每节课的日期、时间、教室、老师、班级 |

## 测试习题

| 集合/表 | 用途 |
|---|---|
| `tests` | 课前测/课后测、文件名、云存储 fileID、批改进度 |
| `test_submissions` | 学生提交或线下完成记录 |
| `grading_records` | 老师手动批改状态、分数、点评 |

## 错题本

| 集合/表 | 用途 |
|---|---|
| `wrong_records` | 学生错题、知识点、错因、订正建议、图片 fileID |
| `wrong_record_images` | 错题照片、压缩图、原图、上传人 |

## 课堂直播

| 集合/表 | 用途 |
|---|---|
| `live_rooms` | 教室直播状态、当前课程、播放地址占位 |
| `live_permissions` | 家长可观看的学生、课程、教室、时间窗 |
| `live_access_logs` | 观看记录、签权时间、过期时间、异常记录 |

## 后续迁移原则

1. 小程序端永远不要直接保存长期有效直播地址。
2. 家长端所有查询都必须按学生绑定关系过滤。
3. 老师端只能查看自己负责班级。
4. 教务端负责录入基础数据和生成邀请码。
5. 文件上传正式版接云存储，数据库只保存 fileID 和元信息。
