# 数据模型草案

本 demo 先使用 `services/mock-db.js` 作为本地数据源。正式上线时建议迁移为微信云开发数据库集合，或独立后端数据库表。

## 账号与手机号

| 集合/表 | 用途 |
|---|---|
| `phone_accounts` | 登录手机号、角色、关联老师/学生/管理员档案 |
| `students` | 学生基础信息、手机号、年级、课程列表 |
| `teachers` | 老师信息、手机号、学科、授课课程 |
| `admins` | 管理员信息和后台权限 |

## 教务与课表

| 集合/表 | 用途 |
|---|---|
| `classrooms` | 15 间教室、设备状态、ClassIn 配置状态 |
| `courses` | 课程班，如“初三生物1班”，绑定老师和学生 |
| `course_sessions` | 课程下的第一次课、第二次课，绑定日期、时间、老师、教室 |
| `assignments` | 可选课前测/课后测记录，不作为第一版主流程 |

## 课后反馈

| 集合/表 | 用途 |
|---|---|
| `lesson_feedbacks` | 老师给学生上传的课后反馈，包含文字、图片 ID、语音 ID |
| `media_files` | 图片和语音元信息、对象存储 key、保留期限、下载权限 |

## 课堂直播

| 集合/表 | 用途 |
|---|---|
| `live_rooms` | 教室直播入口占位、ClassIn 接入状态 |
| `live_permissions` | 可观看课程、学生、课次和时间窗 |
| `live_access_logs` | 观看记录、签权时间、过期时间、异常记录 |

## 后续迁移原则

1. 小程序端不保存对象存储密钥、短信密钥或 ClassIn secret。
2. 学生/家长端所有查询都按手机号绑定的单一学生过滤。
3. 老师端只能查看自己名下课程、课次和学生。
4. 图片和语音正式版接对象存储，数据库保存 `storageKey`、`url`、`retentionUntil` 和元信息。
5. 图片下载、语音播放、ClassIn 入口都由后端或云函数签发临时地址。
