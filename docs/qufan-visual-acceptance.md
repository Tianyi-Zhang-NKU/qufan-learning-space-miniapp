# 趣帆学习空间 小程序视觉验收清单

目标：用微信开发者工具真实预览后，逐页对比 `D:\Desktop\趣帆学习空间` 的视觉语言，确认小程序没有退回普通后台/通用教培模板风格。

## 必验页面

1. 登录页：`pages/login/login`
2. 首页：`pages/home/home`
3. 实时课表：`pages/schedule/schedule`
4. 教室直播：`pages/live/live`
5. 每日习题：`pages/exercises/exercises`
6. 错题本：`pages/wrongbook/wrongbook`
7. 家长端：`pages/parent/parent`
8. 教师端：`pages/teacher/teacher`
9. 机构端：`pages/admin/admin`

## 全局视觉

- 页面背景应为深松绿/暗森林色，不应出现普通白底后台页面。
- 每页应能看到植物线稿/纸纹气质，不能只有纯色渐变。
- 文字主色应为奶油白，弱文字应低对比但可读。
- 页面不应出现蓝紫渐变、纯蓝按钮、营销风大 hero。
- 所有大块卡片应保持半透明玻璃或浅色纸面质感。

## Logo 和头部

- 登录页 Logo 应是“趣帆”文字徽标，不能出现旧图形标识或旧名称。
- 标准功能页头部应有紧凑 Logo、英文 kicker、中文标题、描述。
- 标题不应过大、过粗、过亮，整体应克制。

## 组件一致性

- 背景使用 `z-bg`，不应有页面局部风格不一致。
- Logo 使用 `z-logo`，所有页面标识尺寸应一致。
- 标准功能页头部使用 `z-page-head`。
- 入口卡片使用 `z-tile`，图标应为细线圆形，不是实心彩色块。
- 数字卡使用 `z-metric`，不应像普通数据看板。
- 状态胶囊使用 `z-status`。
- 时间列表使用 `z-list-item`，时间胶囊宽度应稳定。
- 筛选标签使用 `z-chip`。
- 表单字段使用 `z-field`。
- 直播画面使用 `z-camera`，在线点、画面边框、教室文案应统一。

## 自定义 tabBar

- 底部导航应显示深松绿玻璃底。
- 当前页应有奶油绿高亮圆形标识。
- tabBar 不应遮挡页面最后一块内容。
- 切换 首页 / 课表 / 直播 / 错题 时选中态应正确同步。

## 移动端布局

- 375px 宽度模拟器下文字不能挤出卡片。
- 四个 tab 页面底部内容不能被自定义 tabBar 遮挡。
- 直播页两个画面卡在窄屏下不能重叠。
- 浅色纸面卡片里的深色文字应清晰可读。
- 按钮和标签点击态不应出现默认灰块破坏质感。

## 失败判定

出现以下任一情况，不算完成：

- 任何页面仍像普通后台白卡片界面。
- 底部导航是普通原生小程序样式。
- 页面背景没有 趣帆学习空间 的植物线稿/纸纹气质。
- 组件样式在不同页面明显不一致。
- 微信开发者工具有编译错误或自定义组件未渲染。
- 仅代码修改完成但没有截图确认。
